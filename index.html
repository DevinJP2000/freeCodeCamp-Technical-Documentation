Introduction

While most people only notice the website itself, a lot of thought actually goes into making websites accessible to everyone across all abilities and devices. There are many different ways to make a website accessible to everything, this page outlines a few ways to easily make it accessible to many people.

What you already know

This guide assumes you already know...

  • The basics of how websites and HTML files work.
  • How to create a webpage with HTML
  • How to style a webpage with CSS
Make a page for all devices

One easy way to ensure a website works across all devices is with the viewport and content attributes. This lets the website scale content across different device widths.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

While this method does assist in making a page more accessible, there is a way to target certain display widths in CSS with the @media query, which can be editted to only apply to certain device widths or other methods of selecting devices.

@media (prefers-reduced-motion: no-preference)

This specific query alters the webpage so that if a person has a preference for reduced scrolling, the website's scrolling behavior is altered to reflect their preference.

@media (max-width: 768px)

This query alters the logo width depending on the device width that the query reports from the device.

There are a lot of devices to design widths for, luckily there is a handy guide to width specifications for a range of devices here.

Alternate Text

Alternate text helps people who may not otherwise be able to see an image, be able to know what its contents are. This can be helpful if the image does not load, or if the user cannot see it for any reason. I'll show an example down below.

The code for the image element seen above is here.

<img src="https://live.staticflickr.com/7085/7160485639_275957d000.jpg" alternate="A yellow duckling swimming in water.">

The alternate text seen here is "A yellow duckling swimming in water." This text will be seen if the image has not loaded, cannot be loaded, or if the user is using a screen reader.

ARIA Attributes

The ARIA role attribute is used to assign semantic meanings to HTML elements. This is used to guide screen readers or other assistive technologies to guide a user through a webpage. There are many different types, although some types have been replaced by in-built HTML technology, such as the the role "header" being replaced by <h1-6>.

<section role="region" aria-labelledby="student-info">

This is an example of an ARIA role usage. The role of region specifically is used to denote important areas of the document, and should include a more descriptive label detailing why the area of the document is important. The label here is student-info, which denotes that this "region" is for "student info", helping screen readers along.

More can be found about ARIA here on the MDN Web Docs

Access Keys

Access keys can be used by a document to provide key shortcuts across the website. They can be used to provide quicker navigation across a website, especially for those that cannot use a mouse.

<section name="main-section" id="access-keys"> accesskey="6">

This is an example used on this website on how to use an access key. The number 6 is a shortcut to this very section. All sections in this page are numbered, which makes it easier to understand where sections are.

NOTE: BROWSERS OFTEN ADD "ALT" TO THE KEY COMBINATION FOR ACCESS KEYS. MAKE SURE YOU KNOW YOUR BROWSER'S COMBINATION
